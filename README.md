# Web Inspiration Lab

My personal, open-source library of interesting web ideas, interface experiments, component recreations, and full website studies.

This repository is both an inspiration database and a place to write reusable code that may later help improve my portfolio. It is not a single website, so each finished recreation can use the tools that best suit it.

## Library

| Area | Purpose |
| --- | --- |
| [`components/`](components/) | Small, reusable interface recreations |
| [`websites/`](websites/) | Larger page or full-site recreations |

## Workflow

1. Save future inspiration as a GitHub Issue.
2. Create a kebab-case folder under `components/` or `websites/` when ready to build.
3. Implement the idea from scratch and adapt it to my own design preferences.
4. Add a README with a screenshot, credits, setup instructions, and reuse notes.

Different recreations may use different frameworks. Each project folder should contain its own setup instructions and dependencies.

## Public preview

The portfolio is configured to deploy automatically to [GitHub Pages](https://connoer123.github.io/web-inspiration-lab/) after the repository is pushed to GitHub. Localhost links only work on the computer running the development server.

## Suggested GitHub labels

- Type: `component`, `full-website`, `animation`, `layout`
- Progress: `saved`, `ready-to-build`, `building`, `completed`
- Use: `portfolio-idea`, `reusable`

A GitHub Project board can track: **Saved → Ready to build → Building → Completed → Used in portfolio**.

## Responsible recreation

- Implement the idea independently instead of copying proprietary source code.
- Credit and link to the original inspiration.
- Use original or properly licensed text, images, fonts, logos, and other assets.
- Make meaningful visual, content, or interaction changes.
- Do not imply affiliation with the original creator.
- Review the original site's terms and any third-party licenses before publishing.
